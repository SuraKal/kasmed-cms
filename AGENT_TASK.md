# KASMED Content Management Task

## Goal

Build a lightweight Flask + MySQL content management system, a role-aware React
admin panel, and connect the existing public React site to live content without
using Base44.

## Database

- MySQL database: `kasmed`
- Local user: `root`
- Local password: empty
- Every managed item has a UUID, status, sort order, and timestamps.
- Named content uses a readable unique slug.

## Managed Resources

1. Areas of Engagement: name, slug, thumbnail, description, status, sort order.
2. Solutions & Products: name, slug, tags, short description, full description,
   thumbnail, featured flag, status, sort order.
3. Core Values: name, slug, description, icon, status, sort order.
4. Global Suppliers: name, slug, required logo, website, status, sort order.
5. Valued Clients: name, slug, required logo, website, status, sort order.
6. Satisfied Customers: name, slug, required logo, website, status, sort order.
7. Gallery: image, title, alt text, caption, status, sort order.
8. Contact Messages: sender details, message, workflow status, reply text,
   internal notes, and reply date.
9. Site Settings:
   - Address text and map embed URL/iframe.
   - Primary and secondary phones and emails.
   - Business hours.
   - Facebook, LinkedIn, Instagram, and X links.
   - Company name and footer description.

## Administration

- Session-based admin login and logout.
- Dashboard analytics for public page views, clicks, section views, and visitors.
- Super admin, content manager, and contact manager default roles.
- Resource-level role permissions.
- Admin user and role management.
- Image upload support for thumbnails, logos, and gallery images.
- Tabbed admin interface at `/admin`.
- Paginated and searchable content/message management.
- React confirmation dialogs for deletion.
- Filterable audit logs for admin login, logout, create, update, delete, and
  upload actions.
- Slugs and gallery alternative text generated by the backend.

## Public Site

- Load active content from Flask.
- Preserve current hardcoded content as a fallback if the API is unavailable or
  a resource has no database records.
- Submit the contact form to MySQL.
- Apply site settings to contact details, footer, map, and social links.

## Implemented Structure

- `backend/app.py`: Flask application, database models, API routes, permissions,
  uploads, and seed commands.
- `src/admin/config`: resource definitions used by reusable editors.
- `src/admin/components`: separate shell, login, CRUD, settings, messages,
  users, roles, and shared admin controls.
- `src/pages/Admin.jsx`: admin authentication and tab coordination only.
- `src/components/kasmed`: public-facing components only.
- `src/lib/api.js`: shared HTTP and uploaded-asset helpers.

## Local Access

- Public site: `http://localhost:5173`
- Admin panel: `http://localhost:5173/admin`
- Flask API: `http://127.0.0.1:5001`
- Default admin: values from `ADMIN_DEFAULT_USERNAME` and
  `ADMIN_DEFAULT_PASSWORD` in `.env`.

## Public Content Seeder

- Seed missing static content:
  `python -m flask --app backend.app seed-public-content`
- Resync matching records:
  `python -m flask --app backend.app seed-public-content --update`
- Seed definitions: `backend/seeders/public_content.py`
- Seeded totals: 4 engagements, 10 solutions, 4 values, 17 suppliers,
  10 clients, 14 customers, and 7 gallery images.
